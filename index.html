how it works

Three checks. No ground truth required to start.

01 / consistency
Ask it five times
The same prompt gets sent to the model multiple times. If the answers drift, that drift is itself a signal — a model that's certain says the same thing twice.
self-consistency scoring
pairwise similarity diff
02 / grounding
Check it against facts
The model's answer is cross-referenced against a known set of facts you supply. Low overlap means the model may be inventing details that aren't in your source material.
fact coverage scoring
source overlap detection
03 / calibration
Listen to the tone
Models that hedge when uncertain and commit when correct are well-calibrated. Confident language paired with a wrong answer is the most dangerous failure mode there is.
hedge-word detection
confidence-vs-correctness map

drop-in

Three imports. One audit.

No fine-tuning, no labeled dataset to start. Point Sentinel-LLM at any function that returns a string and it runs the full audit.

# run an audit on any LLM call
from sentinel_llm.consistency.auditor import ConsistencyAuditor
from sentinel_llm.grounding.auditor import GroundingAuditor
from sentinel_llm.calibration.auditor import CalibrationAuditor

# point it at your model's query function
ca = ConsistencyAuditor(query_fn)
ca.self_consistency_check(prompt, num_runs=5)
print(ca.get_consistency_score()) # → 59.3

ga = GroundingAuditor()
ga.keyword_overlap_check(answer, known_facts)
print(ga.get_grounding_score()) # → 100.0

cba = CalibrationAuditor()
cba.confidence_language_check(answer, is_correct=True)
print(cba.get_calibration_score()) # → 95.0
